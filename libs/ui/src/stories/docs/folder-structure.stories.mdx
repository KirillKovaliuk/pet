# Структура папок и файлов

Как упоминалось на странице ["Методология дизайна"](/docs/documentation-design-methodology--overview) в проекте
присутствуют 5 видов сущностей для проектирования дизайн-системы. У каждой сущности может быть свой набор файлов,
но можно всегда добавить новый тип. Наиболее используемые будут описаны ниже:

- `index.ts` - файл реэкспорта, используюется для удобного отображения импортов, достаточно указать папку с компонентом
и в импорте будут доступны все экспортируемые данные.
- `*.tsx` - само отображение компонента. Здесь, как правило, содержится бОльшая часть отображения
компонента(часть может находится в `*.container.tsx`).
- `*.declarations.ts` - файл для определения типов\интерфейсов, которые используются во всех остальных файлах компонента.
- `*.container.tsx` - контейнер компонента, служит для получения и обработки данных, которые передаются во вью.
- `*.skeleton.tsx` - файл для скелетонов. Скелетон - компонент, который отображается вместо вью при загрузке данных.
- `*.constants.ts` - файл для констант. Можно сказать, что он является конфигурационным файлом для компонента.
- `*.utils.ts` - файл для утилит, некоторые части компонента могут быть определены в отдельные функции и вынесены из
вью для лучшей читаемости и переиспользования в нескольких местах компонента.
- `*.hooks.ts` - файл для кастомных хуков.
- `*.routes.ts` - файл для роутинга.
- `*.stories.tsx` - файл для документации или песочницы для компонента.
- `*.overview.tsx` - файл для "обзора" компонента, описание и частоиспользуемые кейсы.
- `*.spec.tsx`, `*.container.spec.tsx`, `*.utils.spec.tsx` - файл для тестов.
- `*.module.scss` или `*.styles.ts` - файл стилей. Первое наименование используется в приложениях. Второй в ui-kit для styled-components.

Данные файлы нужны только для локального использования, если какие-то хуки, типы, константы и т.д. используются для нескольких
сущностей, то их стоит выносить в libs/hooks, libs/types, libs/constants соответственно.
