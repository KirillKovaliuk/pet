# Библиотеки и инструменты

### Библиотеки

При выборе библиотек основной упор делался на уменьшении количества зависимых
пакетов, кол-во зависимостей будет указано в скобках.

`react-query(3)` -  это библиотека для управления состоянием в приложениях React, разработанная командой Tanstack.
Она предоставляет интуитивно понятный и мощный способ работы с данными, включая кэширование, инвалидацию и
автоматическую повторную загрузку. React Query облегчает взаимодействие с удаленными данными через API,
улучшая производительность и пользовательский опыт.


`react-hook-form(0)` - это библиотека для управления формами в приложениях React. Она предлагает простой и
мощный подход к управлению состоянием форм, используя хуки React. React Hook Form облегчает процесс валидации
форм, управления их состоянием и сбора данных от пользователей, минимизируя необходимость в шаблонном коде.
Библиотека оптимизирована для производительности и позволяет создавать сложные формы с легкостью, делая процесс
разработки веб-приложений более эффективным.


`yup(4)` - это JavaScript библиотека для валидации данных. Она предоставляет простой и декларативный способ
определения схем валидации данных, которые могут быть применены к различным типам данных, таким как строки,
числа, объекты и массивы. Yup позволяет определять правила валидации с помощью цепочек методов, что делает
код более читаемым и поддерживаемым. Эта библиотека часто используется вместе с другими библиотеками управления
формами в React, такими как Formik и React Hook Form, для обеспечения надежной валидации данных в приложениях React.

`react-18next(2)` - библиотека для интернационализации (i18n) в React-приложениях. React-i18next предоставляет
компоненты и хуки для локализации приложения, позволяя переводить текстовые строки на различные языки и управлять
переключением языков.

`react-imask(2)` - представляет собой библиотеку для React, которая интегрирует IMask.js - мощный инструмент для
маскирования ввода, позволяющий легко добавлять форматирование и валидацию к вводу данных. С помощью React-IMask
мы можем легко создавать и настраивать маски для различных типов данных, таких как номера телефонов, даты, времени и
т. д., обеспечивая удобный и понятный пользовательский опыт при вводе данных в формы. Эта библиотека хорошо интегрируется
с экосистемой React и предоставляет гибкие возможности настройки для удовлетворения различных потребностей в вашем приложении.

`styled-components` - это библиотека для стилизации компонентов в React с помощью использования синтаксиса CSS внутри JavaScript.
Она позволяет создавать компоненты, инкапсулирующие свои стили, что делает код более модульным и переиспользуемым. Стили
определяются непосредственно внутри компонентов с использованием литерала шаблона, что обеспечивает более чистый и
удобочитаемый код. Styled-components также поддерживает передачу пропсов для динамического изменения стилей, что делает
их мощным инструментом для создания динамических пользовательских интерфейсов в React-приложениях.

### Инструменты

`Eslint` - это инструмент статического анализа кода для JavaScript, который используется для идентификации
и исправления паттернов кода, не соответствующих заданным стандартам и правилам стиля. 

`Stylelint` - это инструмент для статического анализа CSS кода, который помогает обнаруживать и исправлять
проблемы со стилями, такие как ошибки синтаксиса, неправильное использование свойств и селекторов, а также
несоответствие определенным стандартам кодирования.

`Commitlint` - это инструмент для проверки сообщений коммитов в репозитории Git на соответствие заданным
правилам и форматам. Он помогает поддерживать стандартные и читаемые сообщения коммитов, что упрощает
работу в команде и отслеживание изменений в проекте.

`Madge` - инструмент для анализа зависимостей в JavaScript-проектах. Madge позволяет
визуализировать и анализировать граф зависимостей между модулями, что помогает понять,
какие части проекта взаимосвязаны и какие модули могут быть удалены или оптимизированы.

`Husky` - инструмент, который позволяет выполнять скрипты или команды перед коммитом или 
пушем в Git-репозиторий. Husky помогает автоматизировать задачи, такие как проверка кода,
запуск тестов или форматирование кода перед сохранением изменений. Запускает вышеперечисленные
инструменты на разных этапах(pre-commit, pre-push).

`PostCSS` - инструмент для обработки и преобразования CSS. PostCSS позволяет использовать
плагины для автоматизации процесса разработки, добавления префиксов для поддержки различных
браузеров, оптимизации стилей и других преобразований CSS.

`Storybook` - инструмент для разработки и документирования компонентов пользовательского
интерфейса. Storybook позволяет создавать и просматривать изолированные компоненты, проверять
различные состояния и взаимодействия компонентов, а также создавать документацию и примеры
использования компонентов.

`NX Workspace` - инструментарий для разработки масштабируемых и модульных приложений.
NX Workspace предоставляет набор инструментов и соглашений, позволяющих эффективно управлять
структурой проекта, управлять зависимостями между модулями и упрощать процесс разработки,
тестирования и развертывания приложений.
